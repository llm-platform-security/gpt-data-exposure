<h3 id="what-is-the-plos-search-api">What is the PLOS Search API?</h3>

<p>The entire PLOS corpus of research articles is regularly indexed on PLOS
servers running <a href="http://lucene.apache.org/solr/">Apache Solr</a>. PLOS
provides programmatic access to the Solr API via the
<strong><a href="http://api.plos.org/search">http://api.plos.org/search</a></strong> endpoint. Documentation for Solr
queries can be found
<a href="https://cwiki.apache.org/confluence/display/solr/The+Standard+Query+Parser">here</a>.
The PLOS endpoint uses “search” instead of the “select” used in the Solr
documentation.</p>

<div class="language-plaintext highlighter-rouge"><div class="highlight"><pre class="highlight"><code>http://api.plos.org/search?q=id:"10.1371/journal.pone.0169045"
</code></pre></div></div>

<h3 id="what-is-solr">What is “Solr”?</h3>

<p><a href="http://lucene.apache.org/solr/">Solr</a> is an open-source
search platform from the Apache
<a href="http://lucene.apache.org/">Lucene</a> project.
Essentially, Lucene is a Java-based indexing and search tool. Solr
provides a friendly way to query Lucene and format the results. PLOS
uses Solr version 4.1.0.</p>

<h3 id="are-there-any-examples-using-the-plos-search-api">Are there any exampleS using the PLOS Search API?</h3>

<p>Two pieces of information are necessary to effectively use the PLOS
Search API. First, being familiar with the
<a href="http://api.plos.org/solr/search-fields/">search fields</a> used to index
documents allows direct searching of specific article sections. Second,
understanding construction of valid <a href="https://cwiki.apache.org/confluence/display/solr/The+Standard+Query+Parser">Solr
queries</a>
using these fields will enable the development of search scripts in the
programming language of choice. Example queries can be found on <strong><a href="http://api.plos.org/solr/examples/">PLOS
Solr Query Examples</a>.</strong></p>

<h3 id="recommended-usage-of-the-plos-search-api">Recommended Usage of the PLOS Search API</h3>

<ul>
  <li><strong>Content Attribution</strong>:  Developers are welcome to make use of the
data returned from the PLOS Search API and we ask that you indicate
that the data originate from PLOS. If you wish to brand your work to
PLOS, you will need to ask us to evaluate the usefulness and
technical capability of your application prior to receiving a PLOS
logo which you may then display on the application.  Please see <a href="http://api.plos.org/api-display-policy/" title="API Display Policy">the
API display
policy</a>
for more information.</li>
  <li><strong>PLOS Search API rate limit</strong>: Please limit your API requests to
7200 requests a day, 300 per hour, 10 per minute and allow 5 seconds
for your search to return results. If you exceed this threshold, we
will lock out your IP address. If you’re a high-volume user of the
PLOS Search API and need more API requests a day, please contact us
at <a href="mailto:api@plos.org">api@plos.org</a> to discuss your options. We currently limit API
users to no more than five concurrent connections from a single IP
address.</li>
  <li><strong>PLOS Search API requests</strong>: Please do not send requests that
return more than 100 rows. That’s a lot of data for our network to
push all at once and it may take some time to return the result set.
If you are getting back a result set that bigger than 100+ rows,
then you likely need to change your query to return a smaller result
set or set a limit on the records returned and page through the
results.</li>
</ul>

<h3 id="i-have-questions-about-using-the-plos-search-api-where-can-i-get-answers">I have questions about using the PLOS Search API. Where can I get answers?</h3>

page</a> for information on the <a href="http://groups.google.com/group/plos-api-developers">PLOS API
