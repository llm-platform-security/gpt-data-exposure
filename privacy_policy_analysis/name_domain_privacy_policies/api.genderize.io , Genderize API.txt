</h1>
  <h2 class="block text-app-text text-4xl font-light font-mono max-w-xl mt-2">
  
    A simple API that predicts the gender of a person based on a first name
  
</h2>
  <div class="grid grid-cols-1 gap-x-8 gap-y-4 mt-8 lg:grid-cols-2">
    <div class="flex flex-col gap-4">
      
    <p class="block text-app-text text-base ">
  The Genderize API allows you to check the statistical probability of a name being male or female.
</p>
    <p class="block text-app-text text-base ">
  The API exposes a single endpoint that accepts a <span class="inline-flex bg-zinc-100 text-sm px-1 py-0.5 rounded text-zinc-600 font-system font-bold border-1 border-zinc-200 mx-[0.15rem] leading-4">name</span> and returns a payload with the prediction. It uses query parameters for requests and returns <a href="http://www.json.org/" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700" target="_BLANK">JSON-encoded</a> responses.
</p>
    <p class="block text-app-text text-base ">
  The API is free for up to 100 names/day. No sign-up or API key is needed. So go ahead and try it out. You can <a href="/register" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700">sign up for an API key</a> if you need more requests.
</p>
    <p class="block text-app-text text-base ">
  We support <a href="#batch-usage" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700">batch processing</a> to reduce the overhead of many HTTP calls.
</p>
    <p class="block text-app-text text-base ">
  The endpoint also supports <a href="#localization" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700">localization</a>, allowing you to scope the gender prediction to a single country for higher accuracy.
</p>
    <p class="block text-app-text text-base ">
  The API is built for high-volume processing, and as such, there is no rate limiting except your daily or monthly limit. You can keep track of that through <a href="#rate-limiting" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700">rate-limiting headers</a> or <a href="/store" class="font-inherit decoration-1 underline-offset-2 text-sky-600 underline hover:text-sky-700">your account</a>.

  <section class="mb-20">
  <span id="fallbacks" class="block text-app-text text-base font-bold font-mono max-w-xl text-brand" data-scrollable>
  
    Input Fallbacks
  
</span>
  <h3 class="block text-app-text text-3xl font-bold font-mono max-w-xl mt-2">
  
    How input fallbacks work
  
</h3>
  <div class="grid grid-cols-1 gap-x-8 gap-y-4 mt-8 ">
    <div class="flex flex-col gap-4">
      
    <p>The API uses a few fallbacks to find a match in our database for cases where the input name is not directly found.</p>
<p>The first fallback will remove all diacritics from the input name and make an attempt to find a match without it. The second fallback will attempt to parse the name in case it's a full name and use the found first name.</p>
<p>So the lookup priority looks like this.</p>
<ul class="list-decimal list-inside">
  <li><p class="inline-block">Direct look up of the input name</p></li>
  <li><p class="inline-block">Look up of the input name with diacritics removed</p></li>
  <li><p class="inline-block">Attemp to parse the name into it's parts and look up the first name</p></li>
</ul>
<p>Even though we support name parsing, it's recommended to always use a first name if you have it available. Naming conventions are different across cultures and contexts and the parsing will never be perfect.</p>
  
    </div>
