<p><strong>Please download and store all data pulled from this API locally</strong> to keep the amount of API calls used to a minimum. Failure to do so may result in either your IP address being blacklisted or the API being rolled back.</p>
<p><strong>Rate Limiting</strong> on the API is enabled. The rate limit is <strong>20 requests per 1 second</strong>. If you exceed this, you are blocked from accessing the API for 1 hour. We will monitor this rate limit for now and adjust accordingly.</p>

<h4 id="card-info"><i class="fas fa-info-circle"></i> Get Card Information</h4>
<p>The Card Information endpoint is available at <code>https://db.ygoprodeck.com/api/v7/cardinfo.php</code></p>
<p>This is the only endpoint that is now needed. You can pass multiple paramters to this endpoint to filter the information retrieved.</p>
<p>The following endpoint parameters can be passed:</p>
<ul>
<li>name - The <strong>exact</strong> name of the card. You can pass multiple <code>|</code> separated names to this parameter (<code>Baby Dragon|Time Wizard</code>).</li>
<li>fname - A fuzzy search using a string. For example <code>&fname=Magician</code> to search by all cards with "Magician" in the name.</li>
<li>id - The 8-digit passcode of the card. You <strong>cannot</strong> pass this alongside name. You can pass multiple comma separated IDs to this parameter.</li>
<li>konami_id - The Konami ID of the card. <strong>This is not the passcode</strong>.</li>
<li>type - The type of card you want to filter by. See below "Card Types Returned" to see all available types. You can pass multiple comma separated Types to this parameter.</li>
<li>atk - Filter by atk value.</li>
<li>def - Filter by def value.</li>
<li>level - Filter by card level/RANK.</li>
<li>race - Filter by the card race which is officially called type (Spellcaster, Warrior, Insect, etc). This is also used for Spell/Trap cards (see below). You can pass multiple comma separated Races to this parameter.</li>
<li>attribute - Filter by the card attribute. You can pass multiple comma separated Attributes to this parameter.</li>
<li>link - Filter the cards by Link value.</li>
<li>linkmarker - Filter the cards by Link Marker value (Top, Bottom, Left, Right, Bottom-Left, Bottom-Right, Top-Left, Top-Right). You can pass multiple comma separated values to this parameter (see examples below).</li>
<li>scale - Filter the cards by Pendulum Scale value.</li>
<li>cardset - Filter the cards by card set (Metal Raiders, Soul Fusion, etc).</li>
<li>archetype - Filter the cards by archetype (Dark Magician, Prank-Kids, Blue-Eyes, etc).</li>
<li>banlist - Filter the cards by banlist (TCG, OCG, Goat).</li>
<li>sort - Sort the order of the cards (atk, def, name, type, level, id, new).</li>
<li>format - Sort the format of the cards (tcg, goat, ocg goat, speed duel, rush duel, duel links). Note: Duel Links is not 100% accurate but is close. Using tcg results in all cards with a set TCG Release Date and excludes Speed Duel/Rush Duel cards.</li>
<li>misc - Pass <code>yes</code> to show additional response info (<a href="#misc-info">details</a>).</li>
<li>staple - Check if card is a <a href="https://db.ygoprodeck.com/search/?&type=Staple">staple</a>.</li>
<li>has_effect - Check if a card actually has an effect or not by passing a boolean true/false. Examples of cards that do not have an actual effect: Black Skull Dragon, LANphorhynchus, etc etc.</li>
<li>startdate, enddate and dateregion - Filter based on cards' release date. Format dates as <code>YYYY-mm-dd</code>. Pass <code>dateregion</code> as <code>tcg</code> <i>(default)</i> or <code>ocg</code>.</li>
</ul>
<p>You can also use the following equation symbols for <strong>atk</strong>, <strong>def</strong> and <strong>level</strong>:<br> "lt" (less than), "lte" (less than equals to), "gt" (greater than), "gte" (greater than equals to).</p>
<p>Examples: atk=lt2500 (atk is less than 2500), def=gte2000 (def is greater than or equal to 2000) and level=lte8 (level is less than or equal to 8).</p>
<p>The specific results from this endpoint are cached for <strong> 2 days (172800 seconds)</strong> but will be manually cleared upon new card entry.</p>
<p id="response-info"><strong>Response Information:</strong></p>
<p>All Cards
<ul>
<li>id - ID or Passcode of the card.</li>
<li>name - Name of the card.</li>
<li>type - The type of card you are viewing (Normal Monster, Effect Monster, Synchro Monster, Spell Card, Trap Card, etc.).</li>
<li>frameType - The backdrop type that this card uses (normal, effect, synchro, spell, trap, etc.).</li>
<li>desc - Card description/effect.</li>
<li>ygoprodeck_url - Link to YGOPRODeck card page.</li>
</ul></p>
<p>Monster Cards
<ul>
<li>atk - The ATK value of the card.</li>
<li>def - The DEF value of the card.</li>
<li>level - The Level/RANK of the card.</li>
<li>race - The card race which is officially called type (Spellcaster, Warrior, Insect, etc).</li>
<li>attribute - The attribute of the card.</li>
</ul>
</p>
<p>Spell/Trap Cards
<ul>
<li>race - The card race which is officially called type for Spell/Trap Cards (Field, Equip, Counter, etc).</li>
</ul>
</p>
<p>Card Archetype
<ul>
<li>archetype - The Archetype that the card belongs to. We take feedback on Archetypes <a href="https://github.com/AlanOC91/YGOPRODeck/issues/10">here</a>.</li>
</ul>
</p>
<p>Additional Response for Pendulum Monsters
<ul>
<li>scale - The Pendulum Scale Value.</li>
</ul>
</p>
<p>Additional Response for Link Monsters
<ul>
<li>linkval - The Link Value of the card if it's of type "Link Monster".</li>
<li>linkmarkers - The Link Markers of the card if it's of type "Link Monster". This information is returned as an array.</li>
</ul>
</p>
<p>Card Sets
<ul>
<li>You can now optionally use <code>tcgplayer_data</code> parameter. Using this will replace our internal Card Set data with TCGplayer Card Set Data.</li>
<li><strong>NOTE:</strong> If using <code>tcgplayer_data</code>, we cannot guarantee that Set Names, Rairites and other info are correct. TCGplayer have occasionally made up Rarity names in the past and don't always conform to correct Card Set Names. A prime example of this is "Legend of Blue Eyes White Dragon" which TCGplayer lists as "The Legend of Blue Eyes White Dragon".</li>
<li>A Card Sets array is now returned as of v5. card_sets</li>
<li>The array holds each set the card is found in. Each set contains the following info: set_name, set_code, set_rarity, set_price.</li>
<li>If using <code>tcgplayer_data</code>, the API will also return <code>set_edition</code> and <code>set_url</code>.</li>
<li>set_price is the $ value.</li>
</ul>
</p>
<p>Card Images
<ul>
<li>A Card Images array is now returned as of v5. card_images</li>
<li>The array holds each image/alt artwork image along with the Card ID. Each set contains the following info: id, image_url, image_url_small, image_url_cropped.</li>
<li>Take this example: <code>https://db.ygoprodeck.com/api/v7/cardinfo.php?name=Decode%20Talker</code></li>
<li>It contains two sets of Card IDs/Images within the card_images array. This is for the default artwork and the additional alternative artwork.</li>
</ul>
</p>
<p>Card Prices
<ul>
<li>A Card Prices array is now returned as of v5. card_prices</li>
<li>The array holds card prices from multiple vendors. <strong>This is the lowest price found across multiple versions of that card</strong>.</li>
<li>cardmarket_price - The price of the card from Cardmarket (in â‚¬).</li>
<li>coolstuffinc_price - The price of the card from CoolStuffInc (in $).</li>
<li>tcgplayer_price - The price of the card from Tcgplayer (in $).</li>
<li>ebay_price - The price of the card from eBay (in $).</li>
<li>amazon_price - The price of the card from Amazon (in $).</li>
</ul>
</p>
<p>Banlist Info
<ul>
<li>A Banlist Info array is now returned as of v5. banlist_info</li>
<li>The array holds banlist information for that card.</li>
<li>ban_tcg - The status of the card on the TCG Ban List.</li>
<li>ban_ocg - The status of the card on the OCG Ban List.</li>
<li>ban_goat - The status of the card on the <a href="https://ygoprodeck.com/an-introduction-to-goat-format/">GOAT Format</a> Ban List.</li>
</ul>
</p>
<p id="misc-info">Additional information returned for <code>misc=yes</code>:
<ul>
<li>beta_name - The Old/Temporary/Translated name the card had.</li>
<li>views - The number of times a card has been viewed in our database (does not include API/external views).</li>
<li>viewsweek - The number of times a card has been viewed in our database this week (does not include API/external views).</li>
<li>upvotes - The number of upvotes a card has.</li>
<li>downvotes - The number of downvotes a card has.</li>
<li>formats - The available formats the card is in (tcg, ocg, goat, ocg goat, duel links, rush duel or speed duel).</li>
<li>treated_as - If the card is treated as another card. For example, Harpie Lady 1,2,3 are treated as Harpie Lady.</li>
<li>tcg_date - The original date the card was released in the TCG.</li>
<li>ocg_date - The original date the card was released in the OCG.</li>
<li>konami_id - The card's Konami ID, if available</li>
<li>has_effect - If the card has an actual text effect. 1 means true and 0 is false. Examples of cards that do not have an actual effect (false/0): Black Skull Dragon, LANphorhynchus, etc etc.</li>
</ul>
</p>
<p><strong>If a piece of response info is empty or null then it will NOT show up. For example, Link Monsters have no DEF, Level or Scale value so those values will not be returned.</strong></p>

<h4 id="api-languages"><i class="fas fa-globe-americas"></i> Endpoint Languages</h4>
<p>We offer the API in the following langugaes: English, French, German, Portuguese and Italian.</p>
<p>Card images are <strong>only stored in English</strong>.</p>
<p>Newly revealed cards leaked from Japan are available only in <strong>English</strong> until the official translation is released.</p>
<p>There are a handful of cards which may not have a translation yet but each language contains over 9000 cards translated. You can reach out to us on <a href="https://github.com/AlanOC91/YGOPRODeck/issues" target="_blank">Github</a> to report any translation misses/issues.</p>
<p>Languages can only be queried on <code>cardinfo.php</code> and must be passed with <code>&language=</code> along with the language code.</p>
<p>The language codes are: <code>fr</code> for French, <code>de</code> for German, <code>it</code> for Italian and <code>pt</code> for Portuguese.</p>
<p>Language Examples:</p>
<ul>
<p><a href="https://db.ygoprodeck.com/card/?search=Survival%20of%20the%20Fittest" target="_blank">Survival of the Fittest</a> in French</p>

<p>Get the 5 latest cards in German</p>
<li><code>https://db.ygoprodeck.com/api/v7/cardinfo.php?language=de&num=5&offset=0&sort=new</code></li>
<p>Get all French cards with "Temps" in the name</p>
<li><code>https://db.ygoprodeck.com/api/v7/cardinfo.php?language=fr&fname=Temps</code></li>
<p>Get all Portuguese cards from the Blue-Eyes archetype</p>
<li><code>https://db.ygoprodeck.com/api/v7/cardinfo.php?language=pt&archetype=Blue-Eyes</code></li>
</ul>

<h4 id="card-images"><i class="fas fa-images"></i> Card Images</h4>
<p><strong>Images are pulled from our image server <code>images.ygoprodeck.com</code>. <strong>You must download and store these images yourself!</strong></strong></p>
<p>Please only pull an image <strong>once</strong> and then store it locally. If we find you are pulling a very high volume of images per second then your IP will be blacklisted and blocked.</p>
<p>Our card images are in <code>.jpg</code> format and are web optimized.</p>
<p>All of our cloud URLs will either be <code>https://images.ygoprodeck.com/images/cards/</code> (full size images), <code>https://images.ygoprodeck.com/images/cards_small/</code> (small size images) or <code>https://images.ygoprodeck.com/images/cards_cropped/</code> (cropped image art). You pass the ID of the card to retrieve the image.</p>
<p>Example Limit Reverse Card Image: <code>https://images.ygoprodeck.com/images/cards/27551.jpg</code></p>
<p>The image URLs are found within the JSON response as <code>image_url</code>, <code>image_url_small</code>, and <code>image_url_cropped</code>, all within the <code>card_images</code> array.</p>
<p>Alternative artwork (if available) will also be listed within the <code>card_images</code> array.</p>
<p>Since v3: Card images are now properly returned without slashes being incorrectly escaped as it was with v2.</p>
</div>
<div class="col-sm">
<figure><img src="https://images.ygoprodeck.com/images/cards/58775978.jpg" style="height: 40%;width: 40%;margin-left: auto;margin-right: auto;display: block;max-width: 100%;border: 3px solid rgba(0, 0, 0, 0.2);" title="Nightmare's Steelcage"><figcaption><i><span class="donatemessage">Card Images</span></i></p></figcaption></figure>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4 id="random-card"><i class="fas fa-random"></i> Random Card</h4>
<p>The Random Card endpoint can be found at <code>https://db.ygoprodeck.com/api/v7/randomcard.php</code>.</p>
<p>This follows the same rate limiting procedures as the card lookup endpoint.</p>
<p>Cache Control is disabled for this endpoint so it should always provide a fresh card.</p>
<p>If any GET parameters are found in the call, then it will return an error.</p>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4 id="card-sets"><i class="fas fa-layer-group"></i> All Card Sets</h4>
<p>The Card Sets endpoint can be found at <code>https://db.ygoprodeck.com/api/v7/cardsets.php</code>.</p>
<p>This follows the same rate limiting procedures as the card lookup endpoint.</p>
<p>This simply returns all of the current Yu-Gi-Oh! Card Set Names we have stored in the database.</p>
<p>This contains the following response info: Set Name, Set Code, Number of Cards and TCG Date (Release Date).</p>
<p>Use this to get a quick snapshot of all the Yu-Gi-Oh! Card Sets sorted by A-Z.</p>
<p>If any GET parameters are found in the call, then it will return an error.</p>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4 id="card-sets-info"><i class="fas fa-layer-group"></i> Card Set Information</h4>
<p>The Card Sets Information endpoint can be found at <code>https://db.ygoprodeck.com/api/v7/cardsetsinfo.php</code>.</p>
<p>This <strong>requires</strong> a parameter of "setcode".</p>
<p>Example usage: <code>https://db.ygoprodeck.com/api/v7/cardsetsinfo.php?setcode=SDY-046</code></p>
<p>This follows the same rate limiting procedures as the card lookup endpoint.</p>
<p>This returns the following information: id, name, set_name, set_code, set_rarity and set_price (in $).</p>
<p>If no (or invalid) GET parameters are found in the call, then it will return an error.</p>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4 id="card-archetypes"><i class="fas fa-dragon"></i> All Card Archetypes</h4>
<p>The Card Archetypes endpoint can be found at <code>https://db.ygoprodeck.com/api/v7/archetypes.php</code>.</p>
<p>This follows the same rate limiting procedures as the card lookup endpoint.</p>
<p>This simply returns all of the current Yu-Gi-Oh! Card Archetype Names we have stored in the database.</p>
<p>Use this to get a quick snapshot of all the Yu-Gi-Oh! Card Archtypes sorted by A-Z.</p>
<p>If any GET parameters are found in the call, then it will return an error.</p>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4 id="code-ver"><i class="fas fa-code-merge"></i> Check Database Version</h4>
<p>The Card Database Version endpoint can be found at <code>https://db.ygoprodeck.com/api/v7/checkDBVer.php</code>.</p>
<p>This follows the same rate limiting procedures as the card lookup endpoint.</p>
<p>This is not a cached endpoint and database_version and date are incremented when:</p>
<ul>
<li>New card is added to the database.</li>
<li>Card information is updated/modified on the main database.</li>
</ul>
<p>If any GET parameters are found in the call, then it will return an error.</p>
</div>
</div>
<div class="row">
<div class="col-sm">
<h4><i class="fas fa-exclamation-triangle"></i> Error Checking</h4>
<p>The main features of v3 is more thorough error checking.</p>
<p>All error codes now return a correct 400 response header.</p>
<p>Almost every parameter will now return an error code if an invalid value is passed to it (as opposed to ignoring it and returning all cards like in previous versions). The user will also be prompted on all the correct possible values to pass so they aren't left guessing.</p>
<p>Here is an example of an invalid value sent through the attribute parameter: <code>https://db.ygoprodeck.com/api/v7/cardinfo.php?type=Effect%20Monster&attribute=wood&num=2</code></p>
<p>The response returned: <code>{"error":"Attribute value of wood is invalid. Please use a correct attribute value. Attribute accepts 'dark', 'earth', 'fire', 'light', 'water', 'wind' or 'divine' and is not case sensitive."}</code></p>
<p>The only way to return all cards now is by having 0 parameters in the request: <code>https://db.ygoprodeck.com/api/v7/cardinfo.php</code>. If invalid parameters are sent, an error will also be returned.</p>
<p>This should save users bandwidth on receiving large requests when requesting malformed urls.</p>
</div>
